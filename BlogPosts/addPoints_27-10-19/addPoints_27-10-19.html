Another new function: addPoints() 27-10-19

I'm on a roll today! Two posts in one day. Here, I'll talk about addPoints() - a new function I've added to my basicPlotteR() R package.

It works in almost the same way as the points() function, but it trys to avoid the points that you're adding overlapping with one another. It uses a lot of the code from my addTextLabels() function.

To get started we'll install my basicPlotteR package and generate some random points:

# Install and load the devtools package
install.packages("devtools")

# Install and load the basicPlotteR package
devtools::install_github("JosephCrispell/basicPlotteR")
library(basicPlotteR)

# Create some random points
n <- 50
coords <- data.frame(X=runif(n), Y=runif(n), Name="Test Label")

Now let's compare the addPoints and points() functions:

#### SET UP PLOT ####

# Set the margins
par(mar=c(1,1,4,1))

# Create a plot of two panels
par(mfrow=c(1,2))

#### points() ####

# Create an empty plot for the points
plot(x=NULL, y=NULL, xlim=range(coords$X), ylim=range(coords$Y), xaxt="n",
  yaxt="n", cex=5, xlab="", ylab="", main="points()")

# Add the points onto the empty plot
points(coords$X, coords$Y, cex=5, col="blue")

#### addPoints() ####
# Plot points and avoid overlapping
plot(x=NULL, y=NULL, xlim=range(coords$X), ylim=range(coords$Y),
  xaxt="n", yaxt="n", xlab="", ylab="", main="addPoints()")

# Add non-overlapping points onto the empty plot
addPoints(coords$X, coords$Y, cex=5, col.line="red", col="blue")

It is a lot of effort for a subtle change! I created the addPoints() function for a plot where I had a lot of overlapping points so it was pretty helpful.
